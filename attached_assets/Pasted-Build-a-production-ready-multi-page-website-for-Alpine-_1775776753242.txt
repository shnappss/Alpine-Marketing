REGIONAL COMPLIANCE LOGIC
Create a configurable compliance system with 3 modes:
1. EU_EEA_MODE
2. SWISS_MODE
3. US_MODE

The site should support region-based behavior via configuration.
Do not hardcode one law only.
Use a conservative default if region cannot be detected.

REGION DETECTION LOGIC
- Use server-side country detection if available from hosting headers or deployment environment.
- If region cannot be reliably detected, default to stricter privacy behavior.
- Do not load marketing/analytics scripts before consent in strict mode.
- All scripts must be managed through a consent layer.

COOKIE / TRACKING LOGIC
Build a consent management UI with:
- Accept all
- Reject non-essential
- Customize settings

Cookie categories:
- Strictly necessary
- Preferences
- Analytics
- Marketing

Rules:
- Essential cookies can load by default.
- Non-essential scripts must be blocked until consent in strict mode.
- The user must be able to reopen settings anytime through “Privacy Choices” in the footer.
- Store consent preferences with timestamp and region mode.
- Build consent-aware wrappers for analytics pixels/scripts.

For safety, configure the site so EU and Switzerland both behave in strict mode by default.
US mode should still support opt-out and privacy controls, and should be easy to switch to stricter behavior if desired.

IMPORTANT:
Do NOT include an obsolete EU ODR platform link.
Do NOT auto-load Meta Pixel, Google Analytics, LinkedIn Insight Tag, Hotjar, or any similar scripts before consent in strict mode.
